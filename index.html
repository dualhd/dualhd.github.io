Dual HD - Horizontal and Vertical video simultaneously

Dual HD frame size is designed for live video streaming, where platform requirements and device orientations vary, and a single frame size doesn't provide the best experience for viewers.

An extra-wide frame size allows creators to stream two different video formats simultaneously, simplifying the process of creating content for multiple platforms.

Use cases include multistreaming to platforms like Twitch/YouTube (horizontal) and TikTok/Instagram (vertical) simultaneously, or creating content for a website with a horizontal video player for desktop users and a vertical video player for mobile users.

Dual HD doesn't specify technical details other than frame size. Modern codecs, like AV1, are recommended though to minimize the growth of bandwidth requirements due to the increased frame size.

Technical specs and templates

Dual HD 2640

Horizontal Full HD and vertical 720p side by side.

  • Frame size (w, h): 2640x1280
  • Horizontal video
    • Size (w, h): 1920x1080
    • Position (x, y): 0, 0
  • Vertical video
    • Size (w, h): 720x1280
    • Position (x, y): 1920, 0

Stream configuration

OBS example
1. Adjust OBS Settings
  • Video -> Base (Canvas) Resolution: 2640x1280
  • Video -> Output (Scaled) Resolution: 2640x1280
  • Output -> Rescale output: Disabled

2. Add Image source to your OBS canvas with the Dual HD template selected. Build your horizontal and vertical layouts on top of the template.

If your processing capacity or bandwidth is insufficient for 2640x1280 streaming, the outgoing stream can be downscaled, provided the receiving service supports scaled Dual HD. A resolution such as 1848x896, which retains 70% of the original width and height, is recommended.

Processing Dual HD video

FFmpeg example

Split Dual HD 2640 video into two destinations with ffmpeg:

 
ffmpeg -i dualhd2640.mp4 \
  -filter_complex "[0:v]crop=1920:1080:0:0[horizontal]; [0:v]crop=720:1280:1920:0[vertical]" \
  -map "[horizontal]" -c:v libx264 fullhd_horizontal_output.mp4 \
  -map "[vertical]" -c:v libx264 720p_vertical_output.mp4
